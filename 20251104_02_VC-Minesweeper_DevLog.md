# 2025년 11월 4일 Minesweeper 개발 일지 (2차)

## 작업 요약

사용자의 피드백을 반영하여 게임의 UI 구조를 근본적으로 변경하고, 기존에 발생했던 문제들을 해결하는 데 중점을 두었습니다. 주요 변경 사항은 UI 레이아웃을 '액자' 형태로 재설계하고, 버튼의 위치 및 동작을 수정한 것입니다.

## 세부 작업 내용

### 1. UI 레이아웃 '액자' 형태로 전면 개편

- **문제점**: 기존의 상단 패널 방식은 게임 창 크기 조절 시 UI 요소(버튼, 텍스트 등)가 잘리거나 위치가 어긋나는 문제가 지속적으로 발생했습니다.
- **해결책**: 게임 보드(타일 영역)의 상하좌우를 감싸는 '액자' 형태의 UI 프레임을 도입했습니다.
    - 상단, 하단, 좌우에 고정된 패널 영역을 확보하여 UI 요소들이 게임 보드와 독립적으로 안정적인 위치를 유지하도록 구조를 변경했습니다.
    - 이제 게임 보드의 크기가 변하더라도 UI 프레임은 유지되므로, UI가 잘리는 문제가 원천적으로 해결됩니다.

### 2. UI 요소 재배치 및 개선

- **버튼 위치 변경**: 사용자의 요청에 따라 'Menu' 버튼과 'Restart' 버튼을 **하단 패널**로 이동시켰습니다.
    - 두 버튼을 하단 중앙에 나란히 배치하여 시각적 안정성과 조작 편의성을 높였습니다.
    - 버튼 위치 계산 로직을 중앙 집중화하여, 창 크기가 어떻게 변하든 항상 중앙 정렬이 유지되도록 수정했습니다.
- **UI 요소 배치**: 시간 및 지뢰 개수 표시는 **상단 패널**에 유지하여 역할을 명확히 분리했습니다.

### 3. 핵심 로직 수정

- **화면 크기 계산**: 새로운 프레임 구조에 맞게 전체 화면 크기가 (게임 보드 크기 + 프레임 두께)로 계산되도록 수정했습니다.
- **게임 보드 위치**: 프레임 내부에 게임 보드가 그려지도록 시작 좌표를 조정했습니다.
- **마우스 좌표 변환**: 사용자의 마우스 클릭 위치가 프레임의 두께를 고려하여 정확한 타일 좌표로 변환되도록 수정했습니다.

## 결론

이번 대대적인 UI 구조 개편을 통해 이전에 발생했던 대부분의 시각적 문제들을 해결하고, 더 안정적이고 확장 가능한 UI 기반을 마련했습니다. 사용자의 명확한 지적 덕분에 더 나은 방향으로 개선할 수 있었습니다.
